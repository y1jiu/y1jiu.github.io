Linux 平均负载

load average: 0.43, 0.68, 0.57
# 三个数字，依次表示过去1分钟、5分钟、15分钟的平均负载（Load Average）

平均负载：指的是单位时间内，系统处于可运行状态和不可中断状态的平均进程数，也称作平均活跃进程数。

**可运行状态的进程：**指的是正在使用 CPU 或者正在等待（排队） CPU 的进程，就是通过 ps 命令查看的处于 R 状态（Running 或 Runnable）的进程。

**不可中断状态的进程：**不可中断状态是系统对进程和硬件设备的一种保护机制。比如一个进程正在对磁盘进行读写操作，为了保证数据一致性（磁盘数据与进程数据一致），在接收到磁盘的回复前，这个进程不能被其他进程中断打断，此时这个进程就处于不可中断状态。

平均负载就是活跃进程数的指数衰减平均值。

平均负载最理想的情况是等于 CPU 个数。查看系统又多少个 CPU 内核在运行：

$ cat /proc/cpuinfo | grep processor | wc -l
6
# 或者
$ cat /proc/cpuinfo | grep 'model name' |wc -l
6

当平均负载高于 CPU 数量的70%~75% 时，操作系统用户体验会开始受到影响，导致进程响应变慢，影响服务正常功能，此时需要分析排查负载高的问题。

最好的办法是需要对系统的平均负载进行监控，根据历史数据，判断平均负载的变化趋势。

平均负载是查看系统整体性能的一个手段，特别需要注意：

  • 平均负载高可能是 CPU 密集型进程导致的
  • 平均负载高不一定代表 CPU 使用率高，可能是 I/O 更繁忙
  • 发现负载高时，可以使用 mpstat 、pidstat 等工具辅助分析负载来源

stress 是一个 Linux 系统压力测试工具，可以模拟平均负载升高的场景。

# 模拟一个CPU使用率100%的场景
$ stress --cpu 1 --timeout 600

# 模拟I/O压力，不停地执行 sync
$ stress -i 1 --timeout 600

# 模拟大量进程，超出CPU运行能力，出现等待CPU的进程
$ stress -c 8 --timeout 600
